Structure:
| Column         | Type                            | Constraints                |
| -------------- | ------------------------------- | -------------------------- |
| id             | SERIAL                          | PK                         |

| solver_id      | INT                             | FK -> users(id)            |
| attempt_number | INT                             | DEFAULT 1                  |
| initiated_at   | TIMESTAMPTZ                     | NOT NULL                   |
| answered_at    | TIMESTAMPTZ                     | NULLABLE                   |

  REOPENED assignments    : 4 logs — attempts 1-3 MISSED, attempt 4 ANSWERED
  base_time = assignment.created_at + 5 minutes

11. issue_history

Structure:
| Column             | Type                                                    | Constraints                          |
| ------------------ | ------------------------------------------------------- | ------------------------------------ |
| id                 | SERIAL                                                  | PK                                   |
| issue_id           | INT                                                     | FK -> issues(id)                     |
| changed_by_user_id | INT                                                     | FK -> users(id), NULLABLE            |
| old_status         | VARCHAR(50)                                             | NULLABLE                             |
| new_status         | VARCHAR(50)                                             |                                      |
| action_type        | ENUM('ASSIGN','UPDATE','COMPLETE','REOPEN','COMPLAINT') |                                      |
| details            | TEXT                                                    | e.g., 'Supervisor verified complete' |
| created_at         | TIMESTAMPTZ                                             | DEFAULT NOW()                        |
| updated_at         | TIMESTAMPTZ                                             | DEFAULT NOW() (Added)                |

Generation rules (applied per issue based on final status):
  Every issue        : NULL -> OPEN            ASSIGN    'Issue created via chat'           created_at=issue.created_at
  Status != OPEN     : OPEN -> ASSIGNED        ASSIGN    'Solver auto-assigned'             created_at=issue.created_at+1h
  Status in {IP,C,RPR,REOPENED,ESC}: ASSIGNED -> IN_PROGRESS UPDATE 'Solver started work'  created_at=issue.created_at+3h
  Status = COMPLETED : RPR -> COMPLETED        COMPLETE  'Supervisor approved'              created_at=issue.updated_at
  Status = REOPENED  : RPR -> REOPENED         COMPLAINT 'Complaint filed - work unsatisfactory' created_at=issue.updated_at-1h
  Status = ESCALATED : IN_PROGRESS -> ESCALATED UPDATE   'System escalated - no response'  created_at=issue.updated_at

================================================================================
ENUM REFERENCE
================================================================================

User Roles         : supervisor, problemsolver, manager
Issue Status       : OPEN, ASSIGNED, IN_PROGRESS, RESOLVED_PENDING_REVIEW, COMPLETED, REOPENED, ESCALATED
Assignment Status  : ACTIVE, COMPLETED, REOPENED
Call Status        : MISSED, ANSWERED
Image Type         : BEFORE, AFTER
AI Flag            : OK, SUSPECT, NOT_CHECKED
Action Type        : ASSIGN, UPDATE, COMPLETE, REOPEN, COMPLAINT
Chat Role          : USER, SYSTEM, AI

================================================================================
FLOW CHAIN
================================================================================

users (supervisor)
  -> issues (raises problem)
     -> issue_assignments (system assigns solver)
        -> call_logs (auto-calls solver)
        -> issue_images (solver uploads fix)
     -> complaints (if not fixed properly)
        -> NEW issue_assignments (re-assign)

Key foreign keys:
  issues.raised_by_supervisor_id    -> users.id
  issue_assignments.assigned_to_solver_id -> users.id
  call_logs.assignment_id           -> issue_assignments.id
  complaints.target_solver_id       -> users.id
  escalations.assignment_id         -> issue_assignments.id
